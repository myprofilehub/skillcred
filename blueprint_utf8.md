# __Why the Hybrid Project Model Works__

The hybrid model solves the core tension in project\-based learning: group projects build teamwork but create identical portfolios, while individual projects build autonomy but miss collaboration skills\. SkillCred's 2\-1\-1 hybrid delivers both\.

__Portfolio Differentiation: How HRs See Each Student__

__What HR Evaluates__

__How the Hybrid Model Delivers It__

Can they code independently?

Solo Projects 1 & 2 ΓÇö entirely built by the student alone, unique code, unique design decisions, unique bugs fixed

Can they collaborate?

Pair Project ΓÇö Git branching, code reviews, task splitting, conflict resolution with a partner, clear module ownership documented

Can they work in a team?

Group Capstone ΓÇö contribution matrix shows exactly which features/modules each student built within the 5\-person team

What's their skill progression?

4 projects arranged beginner ΓåÆ intermediate ΓåÆ advanced ΓåÆ production show a clear growth arc on every portfolio

Are projects real or template?

Mentor\-verified badge \+ unique solo projects \+ PAT viva defense ensure no copy\-paste or template projects pass through

__Contribution Tracking: Making Group Work Transparent__

__Tracking Mechanism__

__How It Works__

Git Commit Analytics

SkillCred auto\-generates per\-student commit graphs ΓÇö lines added/modified, feature branches, PR reviews, and merge history

Module Ownership Map

Each capstone has a documented module map: Student A built auth \+ API, Student B built frontend, Student C built database layer, etc\.

Task Board History

Kanban task assignments tracked throughout the project ΓÇö who picked what, completion timestamps, and dependency chains

Peer Contribution Score

Anonymous peer rating \(1ΓÇô5\) on code output, collaboration, reliability, and initiative ΓÇö averaged and displayed on portfolio

Mentor Verification Notes

Mentor documents individual contributions during code review ΓÇö whose code they reviewed, quality observations per student
